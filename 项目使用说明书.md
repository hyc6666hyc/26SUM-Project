# 《避难所议会》项目使用说明书

> 适用场景：将整个项目文件夹发给同伴，由对方在自己的电脑上安装、运行和测试。  
> 推荐环境：Windows 10/11、Python 3.11 或更新版本。

## 1. 项目简介

《避难所议会》是一个基于 Python 和 Streamlit 的多智能体生存博弈项目，包含：

- AI 自动对局和单人参与模式；
- 普通阵营与隐藏破坏者；
- 事件、讨论、行动、投票、结算和驱逐阶段；
- 动态私人目标，会根据对局天数调整任务次数和数值；
- 规则 Bot 离线运行，以及可选的阿里云百炼 LLM 模式；
- JSON 存档、日志回放和赛后复盘。

## 2. 发送项目前的准备

### 2.1 不要发送私密配置

项目根目录中的 `.env` 可能包含真实 API Key，**不要将它发给其他人**。

发送前建议确认：

- 删除压缩包中的 `.env`；
- 保留 `.env.example`，让同伴自行配置；
- 不要在截图、聊天记录或文档中粘贴 API Key。

### 2.2 建议不打包的目录

以下内容会自动重新生成，体积较大或与本机环境绑定，建议不发送：

```text
.venv/
.pytest_cache/
__pycache__/
work/
output/
data/saves/
.git/              # 如果同伴不需要 Git 历史
```

其中 `.venv` 不应直接复制到另一台电脑，因为里面的路径和二进制文件可能与原电脑绑定。

### 2.3 必须保留的内容

```text
agents/
config/
data/
game/
services/
tests/
ui/
.streamlit/
app.py
main.py
requirements.txt
.env.example
pytest.ini
README.md
项目使用说明书.md
```

整理后，可在 Windows 资源管理器中右键项目文件夹，选择“压缩为 ZIP 文件”再发送。

## 3. 同伴首次安装

### 3.1 安装 Python

先安装 Python 3.11 或更新版本。安装时勾选：

```text
Add Python to PATH
```

安装后打开 PowerShell，检查：

```powershell
python --version
```

如果能显示 Python 版本号，说明安装成功。

### 3.2 解压并进入项目目录

假设项目被解压到：

```text
D:\Projects\FINAL_Project
```

在该文件夹空白处按住 `Shift` 并点击鼠标右键，选择“在终端中打开”。

### 3.3 创建独立虚拟环境

```powershell
python -m venv .venv
```

激活虚拟环境：

```powershell
.\.venv\Scripts\Activate.ps1
```

如果 PowerShell 禁止运行激活脚本，先执行：

```powershell
Set-ExecutionPolicy -Scope Process Bypass
.\.venv\Scripts\Activate.ps1
```

`-Scope Process` 只对当前 PowerShell 窗口有效，不会永久修改系统策略。

### 3.4 安装依赖

```powershell
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
```

如果不想激活虚拟环境，也可直接使用：

```powershell
.\.venv\Scripts\python.exe -m pip install -r requirements.txt
```

## 4. 最快启动方式（不需要 API Key）

项目默认可以完全离线运行，没有 API Key 时使用规则 Bot。

在项目根目录执行：

```powershell
python -m streamlit run app.py
```

如果未激活虚拟环境：

```powershell
.\.venv\Scripts\python.exe -m streamlit run app.py
```

终端会显示类似下列地址：

```text
http://localhost:8501
```

浏览器没有自动打开时，手动复制该地址到浏览器。

## 5. 图形界面使用

### 5.1 开始页

开始页提供两种模式：

- **AI 自动对局**：所有玩家由 AI 或规则 Bot 控制；
- **真人参与模式**：1 名真人和多名 AI 共同游戏。

“高级设置”中可调整：

- 玩家数量；
- 游戏天数；
- 随机种子；
- 自动推进间隔；
- 是否启用隐藏破坏者；
- 是否使用百炼 LLM；
- 使用 LLM 的 Agent 数量。

改变游戏天数后，私人目标和破坏目标会自动缩放。目标数值在新对局建立时确定，不会在游戏中途改变。

### 5.2 AI 自动对局

进入对局后，可以：

- 使用“单步推进”逐阶段观察；
- 使用“自动推进”运行完整对局；
- 在“议会大厅”查看事件、发言和方案；
- 在“私人状态”查看当前视角的目标和线索；
- 在“日志回放”查看公开记录；
- 在“赛后复盘”查看最终阵营和个人结果。

### 5.3 真人参与模式

真人玩家需要按顺序完成：

1. 查看今日事件；
2. 进入讨论，可以使用 3 条推荐发言或编辑自定义发言；
3. 提案、私聊或交易；
4. 执行行动或职业技能；
5. 对公共方案投票；
6. 查看结算并进入下一天；
7. 在可用的驱逐阶段提名、辩护和投票。

投票提交后卡片会保留并置灰，表示投票已锁定，不能重复提交。

## 6. 可选：启用百炼 LLM

离线模式跑通后，再配置 LLM。

先复制环境变量模板：

```powershell
Copy-Item .env.example .env
```

打开 `.env`，填写自己的 Key：

```dotenv
DASHSCOPE_API_KEY=这里填写自己的_API_Key
DASHSCOPE_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
NORMAL_AGENT_MODEL=qwen3.6-flash
STRATEGY_AGENT_MODEL=qwen3.7-plus
REVIEW_MODEL=qwen3.7-plus
```

重新启动 Streamlit，在“高级设置”中打开“使用百炼 LLM”。

建议首次只选择 1 个 LLM Agent，以减少请求次数和费用。模型请求失败时，系统会自动回退到规则 Bot。

## 7. 命令行运行方式

不启动图形界面，直接运行一局规则 Bot：

```powershell
python main.py
```

指定随机种子：

```powershell
python main.py --seed 2026
```

启用 LLM：

```powershell
python main.py --llm --llm-agents 1
```

使用短局测试配置：

```powershell
python main.py --config config/smoke_config.yaml --llm --llm-agents 1 --model qwen3.6-flash
```

命令行模式默认保存：

```text
data/saves/latest_game.json
data/saves/latest_replay.json
```

## 8. 运行测试

安装依赖后执行：

```powershell
python -m pytest -q
```

当前版本正常结果为：

```text
40 passed
```

如果测试全部通过，说明规则引擎、UI 会话、投票、动态目标、存档和基本流程正常。

## 9. 常见问题

### 9.1 `python` 命令找不到

重新安装 Python，并勾选 `Add Python to PATH`。也可在 Windows 中使用 `py` 代替 `python`。

### 9.2 `ModuleNotFoundError`

通常是依赖未安装，或当前未使用项目虚拟环境。重新执行：

```powershell
.\.venv\Scripts\python.exe -m pip install -r requirements.txt
```

### 9.3 PowerShell 禁止运行 `Activate.ps1`

```powershell
Set-ExecutionPolicy -Scope Process Bypass
.\.venv\Scripts\Activate.ps1
```

或者不激活环境，始终使用 `.\.venv\Scripts\python.exe`。

### 9.4 8501 端口被占用

换一个端口启动：

```powershell
python -m streamlit run app.py --server.port 8502
```

然后访问 `http://localhost:8502`。

### 9.5 LLM 不可用

检查：

- `.env` 是否位于项目根目录；
- `DASHSCOPE_API_KEY` 是否为自己的有效 Key；
- 网络是否能访问百炼服务；
- 是否重新启动了 Streamlit。

如果仍不可用，关闭“使用百炼 LLM”即可继续使用规则 Bot。

### 9.6 页面没有显示最新修改

1. 先刷新浏览器；
2. 停止终端中的 Streamlit（`Ctrl+C`）；
3. 重新执行 `python -m streamlit run app.py`；
4. 规则或目标只会应用到新建立的对局，必要时返回开始页重新开局。

## 10. 项目目录说明

```text
FINAL_Project/
├─ agents/          # 规则 Bot、LLM Agent、真人玩家接口
├─ config/          # 对局参数和短局配置
├─ data/            # 事件、私人目标和 JSON 存档
├─ game/            # 核心规则、状态机、投票、驱逐和计分
├─ services/        # LLM 客户端、存储和日志
├─ tests/           # 自动化测试
├─ ui/              # Streamlit 开始页、仪表盘和真人操作界面
├─ app.py           # 图形界面入口
├─ main.py          # 命令行入口
├─ requirements.txt # Python 依赖
└─ .env.example     # 环境变量模板
```

## 11. 推荐的验收流程

同伴收到项目后，按以下顺序验收：

1. `python --version` 能显示版本；
2. 成功创建 `.venv`；
3. `pip install -r requirements.txt` 无错误；
4. `python -m pytest -q` 显示 `40 passed`；
5. `python -m streamlit run app.py` 能打开开始页；
6. 先关闭 LLM，完整运行一局规则 Bot；
7. 如需要，再填写自己的 API Key 并测试 1 个 LLM Agent。
